## PROMPT 5: AI INTEGRATION (Replicate API)

### **Copy-Paste This:**

```
Integrate AI virtual try-on using Replicate API:

1. Setup:
   - Install replicate package
   - Add REPLICATE_API_TOKEN to environment variables
   - Use IDM-VTON model: "cuuupid/idm-vton"

2. Backend Route (app.py):
   - Add route: POST /api/tryon
   - Accept: human_image filename, dress_image filename
   - Read both images and convert to base64
   - Call Replicate API with:
     * human_img: base64 data URI
     * garm_img: base64 data URI
     * garment_des: description
     * is_checked: True
     * denoise_steps: 30
     * seed: 42
   - Save result image to uploads/results/
   - Return result image path

3. Image Processing:
   - Resize images if larger than 1536x1536 (use Pillow)
   - Handle API response (URL or bytes)
   - Save result with unique filename

4. Frontend Updates:
   - Add "Generate Try-On" button
   - Disable button until both images selected
   - Show loading indicator during processing
   - Display result image when ready
   - Add download button for result

5. Error Handling:
   - No API key configured
   - API request fails
   - Invalid images
   - Processing timeout

Code example for Replicate call:
```python
import replicate

output = replicate.run(
    "cuuupid/idm-vton:...",
    input={
        "human_img": human_data_uri,
        "garm_img": dress_data_uri,
        "garment_des": "clothing item",
        "is_checked": True,
        "denoise_steps": 30,
        "seed": 42
    }
)
```

Important: Handle both URL and FileOutput responses from API.
Add timeouts and retry logic.
```

**Expected Result:** Working AI try-on generation
